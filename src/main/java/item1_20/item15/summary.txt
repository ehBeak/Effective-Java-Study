접근 제한자: 정보 은닉의 핵심 장치

접근 제한자 사용법
1. 모든 클래스와 멤버의 접근성을 가능한 한 좁혀야 한다.
2. public일 필요가 없는 클래스의 접근 수준을 package-private 톱레벨 클래스로 좁힌다.2
3. public클래스의 인스턴스 필드는 되도록 public이 아니어야 한다.
4. public클래스는 상수용 public static final 필드 외에는 어떠한 public 필드도 가져서는 안된다.

접근 제한자 조건
1. 톱 클래스에와 인터페이스는 package-private과 public만을 부여할 수 있다.
